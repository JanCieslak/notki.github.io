Intro

Do poszerzania wiedzy jak zawsze zachęcam: w3school, możecie także rozszerzać notki jak będziecie uważać, że jest coś wartego uwagi, bądź zapisania na przyszłość.

Identyfikatory

Identyfikatory są sposobem na oznaczanie tagów HTML do późniejszego wykorzystania przez CSS-a lub JS-a.

<!-- id - identyfikator unikalny na całą stronę, powinien opisywać dany element,
          np. 'website-title' może opisywać tytuł strony w navbarze -->
<h1 id="website-title"></h1>

<!-- class - to identyfikator, który może się powtarzać,
             element może mieć więcej niż jedną klasę, oddzielamy je wtedy spacją, 
             klasy zazwyczaj opisuje to co robi / dodaje do danego elementu,
             np. 'padding-3' to nasza klasa dodająca jakiś padding (tutaj zależy co oznacza 3) do naszego button-a -->
<button class="margin-1 padding-3"></button>

<!-- możemy używać obu identyfikatorów na raz -->
<button id="unique-identifier" class="my-button big-text"></button>

Identyfikatory definiujemy, aby potem ich użyć w plikach CSS, bądź Javascript

<button id="click-me-button" class="border-0 padding-3">Click me</button>

mając taki element html możemy go użyć w CSS-ie następująco

.padding-3 {
    padding: 30px
}

.border-0 {
    border: none;
}

#click-me-button {
    margin: 50px;
}

bądź w JS-ie wypisać w konsoli ‘click’ jeśli użytkownik kliknie dany przycisk

document.querySelector('#click-me-button').onClick = (event) => {
    console.log('click')
}

Jeśli chodzi o klasy CSS-owe istnieją biblioteki, które gromadzą klasy z najczęstszymi wymaganiami od CSS-a, aby pisać go jak najmniej, tylko przy pisaniu HTML-a używamy już tych klas, które pozwalają nam na szybsze tworzenie tagów o dobrych rozmiarach, pozycji i kolorze. Oto kilka z nich:

  • Tailwind
  • Bootstrap
  • Tachyons

Selectory

Selectory to wyrażenia, które pozwalają nam zaaplikować dane style do tagów opisem pasujących do tych wyrażeń. Jest kilka typów selectorów i opiszę te najważniejsze poniżej, a na koniec tego rozdziału dam linki do innych selectorów (takich bardziej fancy).

Type selectory - szukanie tagów po typie

/* zmatchuje się ze wszystkimi tagami 'a' */
a {
    background-color: red;
}

/* zmatchuje się ze wszystkimi div-ami */
div {
    background-color: blue;
}

Id selectory - szukanie tagów po id-kach

/* zmatchuje się z tagiem o id title */
#title {
    padding: 15px;
    border: 1px solid black;
    margin: 10px;
}

Class selector-y - szukanie tagów po klasach

/* zmatchuje się z każdym tagiem posiadającymi klasę border-3 */
.border-3 {
    border: 1px solid red;
}

/* zmatchuje się z każdym div-em posiadającym klasę border-3 */
div.border-3 {
    border: 1px solid red;
}

/* zmatchuje się z tagami h1, tylko którymi posiadają klasy border-3 i padding-3 */
h1.border-3.padding-3 {
    border: 1px solid red;
}

Combos-y

Sąsiad (+)

/* zmatchuje się dla każdego przypadku gdzie tag p występuje po tagu img i należą do tego samego parent-a */
img + p {
}

Dziecko (>)

/* zmatchuje się z li, które jest dzieckiem ul z klasą my-things */
ul.my-things > li {
  margin: 2em;
}

Przodek ( )

/* zmatchuje się z li, jeśli tag ul z klasą my-things jest przodkiem (parent, parent's parent, parent's parent's parent, etc.) tagu li */
ul.my-things > li {
    margin: 2em;
}

Combo (,)

/* zmatchuje się z każdym tagiem span i div i zaaplikuje dla nich border 2px red solid */
span,
div {
  border: red 2px solid;
}

Pseudo-klasy i pseudo-elementy

Pseudo elementy to już bardziej zaawansowany temat, ale ofc polecam poczytać, szczególnie najczęściej widziałem użycie elementów ::before i ::after. Pseudo klasy są prostsze do zrozumienia i częściej używane. Przykładem jednej z najczęściej używanej klasy byłoby :hover (czyli klasa, która wskazuje na stan elementu przy najechaniu na niego myszką). Możemy w ten sposób zrobić element div, gdzie będzie miał tło czerwone, a po najechaniu na niego myszką będzie miał tło niebieskie.

div {
    background-color: red;
}

div:hover {
    background-color: blue;
}
