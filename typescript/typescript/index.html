=======

Typescript jest to język zbudowany na Javascipcie. Zaletą takiego rozwiązania jest to, że Typescirpt może używać tych samych rzeczy (bibliotek) co Javascript przy czym dając jego zalety. Do zalet można głównie zaliczyć typy, ale Typescript daje nam o wiele więcej. Konfiguracja Typescripta znajduje się w pliku tsconfig.json. Definiuje on w jaki sposób możemy używać Typescripta i do jakiej formy Javascripta się skompiluje.

Typy (Types)

Jedną z wad (bądź zalet) w Javascripcie jest to że nie musimy podawać typu - dynamicznie typowany język. Typescript rozwiązuje to dodając typy. Żebyście zrozumieli dlaczego typy są ważne dam przykład z funkcją.

// kod Javascript

function addTwenty(input) {
  return input + 20
}

console.log(addTwenty(20)) // returns 40 - number
console.log(addTwenty('20')) // returns '2020' - string

Jak widać fukncja dodaje 20 do tego co przyjmuje. Jeśli przyjmie liczbę doda do niej 20 i zwróci, ale jeśli wrzucimy do niej coś czego autor funkcji zapewne nie chciał, np. string. To wtedy funkcja zwróci nam stringa, w tym przypadku ‘2020’, ponieważ my wrzucamy ‘20’ a Javascript próbując dodać tekst do liczby, zamienia liczbę w tekst automatycznie i dodaje tekst do tesktu, dlatego ‘2020’. Dzięki Typescriptowi, możemy temu zapobiec nadając funkcji typy, aby użytkownik tej funkcji wiedział jak jej używać.

// kod Typescript

function addTwenty(input: number): number {
  return input + 20
}

console.log(addTwenty(20)) // returns 40 - number
console.log(addTwenty('20')) // Error: type string when number was expected

Jak widzicie na przykładzie dodałem 2 annotacje (oznaczenia) typów, jeden dla parametru input, że ma mieć typ number i na końcu funkcji mówiący o tym co ta funkcja będzie zwracać.

Mamy wiele typów, które każdy musi znać. Będzie ich więcej w przyszłości, ale na razie ograniczymy sobie ten zbiór na dla prostoty.

let liczba1: number // jeśli od razu nie przypisujemy wartości przypisujemy od razu typ
liczba1 = 123
const liczba2 = 123 // jeśli prawa strona = jest znana to wtedy możemy pominąć typ (Typescript go raczej zgadnie)
const liczba3: number = 123 // dla pokazania typów ja jednak będę je pisał, żebyście wiedzieli jaka prawa strona = da jaki typ
const hex   : number = 0xff // 255 w systemie szesnastkowym
const binary: number = 0b100 // 4 w systemie binarnym

const color1: string = 'blue'
const color2: string = "red" // stringi mogą być pisane za pomocą pojedyńczych lub podwójnych cudzysłowiów lub ``
const colors: string = `${color1} ${color2}` // `` daje nam tak zwane string interpolation, dzięki której możemy zawierać inne zmienne w tekście (nie tylko inne stringi)
const multiLine: string = `
- one
- two
- three
` // w `` stringu możemy też pisać wielo wierszowe stringi (nie można tego robić za pomocą cudzysłowiowych stringów)

const isDone1: boolean = true
const isDone2: boolean = false

Na razie wystarczy, jest ich o wiele więcej i polecam w wolnym czasie dowiadywać się jak najwięcej o nich. Tutaj macie linka do stroki z najważniejszymi typami

Operatory

Operatory są funkcje, które zachodzą pomiędzy zmiennymi, liczbami, stringami i innymi typami. Jest ich parę typów, ale przedstawimy sobie te najważniejsze, ale odsyłam tutaj po więcej info.

Operatory arytmetyczne

Są to operatory to wykonywania obliczeń matematycznych

// Dodawanie (+)
const dodawanie = 2 + 2
console.log(dodawanie) // 4

// Odejmowanie (-)
const odejmowanie = 2 - 2
console.log(odejmowanie) // 0

// Mnożenie (*)
const mnozenie = 2 * 2
console.log(mnozenie) // 4

// Reszta z dzielenia (%)
const reszta = 5 % 2
console.log(reszta) // 1

// dzielenie (/)
const dzielenie = 5 / 2
console.log(dzielenie) // 2

// Inkrementacja (++)
let inkrementacja = 2
inkrementacja++
console.log(inkrementacja) // 3

// Dekrementacja (--)
let dekrementacja = 2
dekrementacja--
console.log(dekrementacja) // 1

Operatory relacji

Operator, który na podstawie podanych argumentów zwraca wartość boolean (true/false), w zależności czy dana operacja spełnia warunek operatora.

// Większy niż
const wiekszy = 20 > 10
console.log(wiekszy) // true

// Mniejszy niż
const mniejszy = 10 < 20
console.log(mniejszy) // true

// Większy bądź równy
const wiekszyLubRowny = 10 >= 10
console.log(wiekszyLubRowny) // true

// Mniejszy bądź równy
const mniejszyLubRowny = 10 <= 10
console.log(mniejszyLubRowny) // true

// Równy
const rowny = 10 === 10
console.log(rowny) // true

// Nierówny
const nieRowny = 11 !== 10
console.log(nieRowny) // true

Operatory logiczne

Operatory działające na wartościach boolean, łączące 2 lub więcej warunków. Po więcej kombinacji zapraszam tutaj.

// i (&&) - jest prawdą jeśli obie strony są true
const and = 2 > 1 && 4 > 2 // true i true = true
const and2 = 2 > 1 && 4 < 2 // true i false = false

// lub (||) - jest nieprawdą jeśli obie strony są false
const or = 2 > 1 || 4 > 2 // true lub true = true
const or2 = 2 > 1 || 4 < 2 // true lub false = true

// zaprzeczenie (!) - odwraca wartość boolean (z true na false i z false na true)
const not = !(2 > 1) // zaprzeczenie true = true
const not2 = 2 > 1 && !(2 < 1) // true i zaprzeczenie false = false

Operatory przypisania

// Przypisanie (=)
const a = 1 // zwykłe przypisanie wartości do zmiennej

// Reszta operatorów to ułatwienia, które zastępują dłuższy zapis

// Dodaj i przypisz (+=)
let b = 2
b += 2
console.log(b) // -> 4
b = b + 2 // jak widzicie b += 2 jest uproszczeniem dla zapisu po lewo, czyli b = b + 2
console.log(b) // -> 6


// Odejmij i przypisz (-=)
let c = 2
c -= 2
console.log(c) // -> 0

// pomnóż i przypisz (*=)
let d = 2
d *= 2
console.log(d) // -> 4

// podziel i przypisz (/=)
let e = 2
e /= 2
console.log(e) // -> 1

Tenary operator

Jest to trochę trudniejszy operator, ale na pewno przydatny. Jest to skrót dla if-a i przypisania wartości. Załóżmy, że chcemy coś przypisać do wartości jeśli warunek jest spełniony, a jeśli nie to coś innego. Podaję przykład

const age = 16

let isAdult: boolean

if (age > 18) {
  isAdult = true
} else {
  isAdult = false
}

console.log(isAdult) // false

Dzięki tenary operator możemy przypisać isAdult w jednej linijce.

Szablon

const result = /* warunek */  ?  /* wartość jeśli true */  :  /* wartość jeśli false */

Rozwiązanie

const age = 16
const isAdult = age > 18 ? true : false

// Możemy także wstawiać wszystko w pola wartość i rozbudowywać warunki,
// ale zasada tego operatoa jest taka żeby go używać, jeśli będzie krótka
// nie zaleca się robić długich operacji na tenary operatorze
const ageName = age > 18 ? "dorosły" : "nieletni"

const hasCar = true
const isAdultAndHasCar age > 18 && hasCar === true ? true : false

Zauważcie że dzięki temu mogłem zrobić isAdult jako const co jest bardzo ważne, ponieważ zmniejsza to ryzyko, że gdzieś później w programie ta wartość zostanie zmieniona, bądź nadpisana.

String operator

Konkatenacja stringa (łączenie obu tekstów)

const hello = "hello"
const world = "world"
const helloWorld = hello + " " + world
console.log(helloWorld) // "hello world"
console.log(hello + " world") // "hello world"

Zmienne / Stałe (Variables / Constants)

Już powinniście wiecieć, że zmienne to wartości, które mogą się zmienić w trakcie działania naszego programu (i jego “życia” - o tym zaraz) i tworzymy je przy pomocy let, a stałe są to wartości nie zmieniające się w trakcie działania programu (i jego “życia”). Zmienne i stałe tworzymy w ten sposób

/* modyfikator (let/const) nazwa zmiennej = jego wartość, np. */
const age = 16

Zakres długości życia zmiennych / stałych (Variable Scopes)

Do zmiennych i ich scope-ów polecam poczytać tutaj W scope-ach, chodzi o to że dana zmienna może być użyta w zależności od tego jaki jest jej, scope i od niego zależy żywotność danej zmiennej.

if (20 > 10) { // początek scope-u
  const abc = 123
  console.log(abc) // -> 123
} // koniec scope-u (wszystkie wartości w scop-ie zostają uwolnione z pamięci i nie możemy ich więcej używać)
console.log(abc) // Error: Nie ma takiej zmiennej jak abc

W powyższym przykładzie widzimy, że abc ma scope if-a, czyli może być używany tylko w if-ie. Scope zazwyczaj jest oznaczany klamrami {} (używanych, w np. if, for, while, do..while, function), a żywotność danej zmiennej kończy się z } klamrą, a pamięć naszego programu zmniejsza się o daną zmienną (ponieważ nasz komputer nie potrzbuje trzymać jej w pamięci). Może to być dla wasz cięzki koncept, ale polecam zajrzeć tutaj po bardziej szczegółowe i przygotowane wytłumaczenie.

Daję jeszcze przykład, którego chyba nie ma w linku i jest to tworzenie scope-u za pomocą {}

const a = 1

if (true) { // definiujemy scope
  const b = 2

  { // definiujemy scope
    const c = 3

    console.log(a)
    console.log(b)
    console.log(c) // Error: nie ma takiej zmiennej jak b
  } // koniec scope-u (umiera 'c' :<)

  console.log(a)
  console.log(b)
  console.log(c) // Error: nie ma takiej zmiennej jak b
} // koniec scope-u (umiera 'b' :<)

console.log(a)
console.log(b) // Error: nie ma takiej zmiennej jak b
console.log(c) // Error: nie ma takiej zmiennej jak b

Konwersje typów (Type conversions)

Na razie najważniejsze konwersje to string => number i number => string

// string => number
const num = 123
const stringNum = num.toString()
// lub
const stringNum2 = String(num)

// number => string
const str = '123'
const number = Number(str)
// lub
const number = parseInt(str)

// jeśli mamy liczbę zmiennoprzecinkową w stringu możemy użyć parseFloat, inaczej wszystko po przecinku zostanie zaokrąglone
const realString = '123.123'
const integer = parseInt(realString) // => 123
const real = parseFloat(realString) // => 123.123

Pętle (Loops)

Pętle uławiają nam zazwyczaj w sytuacjach gdzie musimy wykonać daną akcję x razy przy spełnieniu danego warunku. Na przykład jeśli chcemy przeczytać plik tekstowy linijka po linijce, możemy użyć do tego pętli. Najpierw oteramy plik (inicjalizacja) i czytać plik dopóki (warunek) nie dojdziemy do końca pliku, a w każdej iteracji pętli przechodzić do następnej linii (inkrementacja). Typów pętli jest parę, każda z nich mogłaby być zastąpniona jedną (nie ważne jaką), ale jest ich parę ponieważ w zależności od wymagań, jedna może być bardziej dostosowana do naszego problemu niż druga.

for

Szkielet

for (/* inicjalizacja */; /* warunek */; /* inkrementacja */) {
    // kod wykonujący się w pętli tak długo jak warunek zostaje spełniony (pętla)
}

Na przykład

for (let n = 0; n < 10; n++) {
  if (n % 2 == 0) {
    console.log(`${n} to liczba parzysta`) // -> 0, 2, 4, 6, 8
  }
}

while

Szkielet

while (/* warunek */) {
    // kod wykonujący się w pętli tak długo jak warunek zostaje spełniony (pętla)
}

Na przykład

let number = 0
while (number < 10) {
    number++ // zwiększamy number o 1 na koniec każdej pętli
}
console.log(number) // -> 9

do while

Szkielet

do {
    // kod wykonujący się w pętli tak długo jak warunek zostaje spełniony (pętla)
} while (/* warunek */)

Na przykład

let number = 0
do {
    number++ // zwiększamy number o 1 na koniec każdej pętli
} while (number < 10) 
console.log(number) // -> 10

Funkcje

Funkcji używamy do reużycia danej funkcjonalności. Na przykład robimy grę kółko i krzyrzyk i potrzebujemy funkcji która wykona turę, ponieważ tury są wykonywane co ruch, warto zawrzeć tą funkcjonalność w funkcji. Kolejnym przykładem z gry może być funkcja sprawdzająca czy któryś z graczy wygrał.

// funkcje możemy pisać na dwa sposoby

// to jest tradycyjny sposób
function sayHello() {
  console.log('Hello')
}

// tutaj przypisujemy anonimową funkcję do wartości sayHello2
const sayHello2 = () => {
  console.log('Hello2')
}

// funkcje wywołujemy w ten sposób
sayHello() // prints 'Hello'
sayHello2() // prints 'Hello2'

// funkcje mogą przyjmować argumenty i zwracać wartości (wtedy podajemy ich typy)
// jeśli nic nie zwracamy to nie musimy nic pisać
const say = (message: string): string => {
  console.log(message)
  return message // wartości zwracamy za pomocą return <wartość>
}

// Return gdy nic nie chcemy zwracać
function earlyReturn(value: boolean) {
  if (value) {
    // Jeśli nie zwracamy wartości możemy wypisać return i oznacza to to 
    // że wychodzimy w funkcji i nic dalej się w niej nie dzieje
    return
  }

  console.log('Hello')
}

// Więcej niż 1 return - możemy definiować ich więcej jeśli mam rozgałęzienia jak ify bądź pętle
function moreReturns(value: boolean, message: string): string {
  if (value) {
    return 'true value ' + message
  }
  return 'false value ' + message
}

// Co ciekawe możemy przypisywać funkcje do zmiennych (2 wersja funkcji), ale też przyjmować je jako argumenty
// co pozwala na ciekawe rozwiązania na różne problemy i w wielu przypadkach pozwala na stworzenie
// bardzo elastycznej funkcji. Możemy na przykład stworzyć funkcję calculate, która w zależności, od tego jaką operację
// przyjmię taki będzie wynik końcowy. Tworzymy funkcję która przyjmuje 2 liczby i funkcję, która też przyjmuje 2 liczby
// i zwraca liczbę (nasza funkcja calcuate też zwraca liczbę).
const calculate = (a: number, b: number, operation: (x: number, y: number) => number): number {
  // wywołujemy ją jak normalną funkcję, parametry a i b są tylko po to żeby je przekazać dalej
  return operation(a, b)
}

// potem możemy sobie definiować funkcje spełniające definiję funkcje (number, number): number, na przykład
// nazwy parametrów nie muszą się zgadzać (x, y), ale typy jak najbardziej
const add = (a: number, b: number): number => {
  return a + b
}

const sub = (a: number, b: number): number => {
  return a - b
}

// A nasze calculate używamy w następujący sposób
const a = 15
const b = 5
const result = calculate(a, b, add) // podajemy funkcję bez (), nawiasy oznaczają wywołanie a bez zwykłą wartość
const result2 = calculate(a, b, sub)

console.log(result) // -> 20
console.log(result2) // -> 10

// Możemy też używać tak zwanych wyrażeń Lambda, czyli skróconych funkcji
const result3 = calculate(a, b, (x, y) => x + y) // to samo co z add

// Możemy też zwracać funkcję
const returnLogger = () => (any) => {
  return console.log
}

returnLogger()('Hello') // pierwszy () zwraca console.log a drugi () wywołuje console.log z argumentem 'Hello'

// Funkcje rekurencyjne, jest to funkcja która wywołuje samą siebie, na przykład
function count(n: number, max: number): number {
  if (n >= max) {
    return n
  }

  return count(n + 1, max)
}

// jeśli podamy n = 0 i max = 2, wtedy
const result = count(0, 2) // = 2
// wewnątrz
// count(0, 2)
//  count(1, 2)
//   count(2, 2) - wypełnia ifa i zwraca 2 i wcześniejsze funkcje zwracają to co ta czyli 2

// jak zawsze po detale odsyłam do internetów
